================================================================================
ENDPOINTS & FUNCTIONALITY
================================================================================

POST /chat
   Input: { "message": "user query" }
   Output: AI response with confidence, source, intent
   - Checks knowledge base first
   - Falls back to LLM if no match
   - Escalates if confidence too low

GET /health
   Output: System status, LLM availability, model info
   - Quick health check
   - Used by UI status indicator
   - Used by Docker health checks

POST /support-trigger
   Input: { "user_message": "...", "reason": "..." }
   Output: Support ticket ID and confirmation
   - Manual escalation endpoint
   - Creates support ticket
   - Returns ticket ID for tracking

GET /support/tickets
   Output: List of escalated support tickets
   - View recent escalations
   - Check confidence scores
   - Review escalation reasons

GET /support/stats
   Output: System statistics
   - Total escalations
   - Escalation breakdown by reason
   - Average confidence scores

GET /knowledge-base
   Output: All knowledge base items
   - View current Q&A database
   - Check keywords for each answer

POST /knowledge-base/add
   Input: { "intent": "...", "answer": "...", "keywords": [...] }
   Output: Success confirmation
   - Add new answers via API
   - Build knowledge base programmatically

CONFIGURATION OVERVIEW
================================================================================

Environment variables (.env):

OLLAMA_BASE_URL
   - URL where Ollama runs
   - Default: http://localhost:11434

OLLAMA_MODEL
   - Which model to use
   - Options: llama2, mistral, neural-chat, dolphin-phi
   - Default: llama2

CONFIDENCE_THRESHOLD
   - Minimum score to return AI response (0-100)
   - Below threshold → escalate to support
   - Default: 70 (Good balance)

DB_TYPE
   - Database backend
   - Options: json (simple), sqlite (production)
   - Default: json

SERVER_HOST
   - Where server listens
   - Default: 127.0.0.1 (localhost only)
   - Use 0.0.0.0 for network access

SERVER_PORT
   - Which port to run on
   - Default: 8000

LOG_LEVEL
   - Verbosity of logging
   - Options: DEBUG, INFO, WARNING, ERROR
   - Default: INFO

SUPPORT & TROUBLESHOOTING
================================================================================

Check logs:
  - Terminal where backend is running (real-time logs)
  - data/support_log.json (escalation log)
  - View via /support/stats endpoint

Common issues:
  1. "Cannot connect to API"
     → Make sure backend is running
     → Check port 8000 is available
  
  2. "Ollama offline"
     → Start Ollama: ollama serve
     → Check: curl http://localhost:11434/api/tags
  
  3. "Slow responses"
     → First response loads model (normal)
     → Use smaller model: ollama pull neural-chat
     → Check available RAM
  
  4. "Port in use"
     → Change SERVER_PORT in .env
     → Or: lsof -ti:8000 | xargs kill -9

Full troubleshooting in SETUP.md
