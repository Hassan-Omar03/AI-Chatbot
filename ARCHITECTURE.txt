5. DATA FLOW ARCHITECTURE
================================================================================

USER INPUT
    │
    └──→ [Frontend HTML/JS]
         │
         └──→ Fetch API (JSON)
             │
             └──→ [FastAPI Backend]
                  │
                  ├─→ [Intent Detector]
                  │   └──→ Intent classification
                  │
                  ├─→ [Knowledge Base]
                  │   ├──→ if FOUND: Return (100% confidence)
                  │   └──→ if NOT FOUND: Continue
                  │
                  ├─→ [LLM Engine]
                  │   ├──→ Send to Ollama HTTP API
                  │   ├──→ Receive response
                  │   └──→ Extract confidence
                  │
                  ├─→ [Safety System]
                  │   ├──→ Evaluate confidence
                  │   ├──→ if HIGH: Return response
                  │   └──→ if LOW: Escalate
                  │       └──→ Create support ticket
                  │           └──→ Log to data/support_log.json
                  │
                  └──→ Format JSON response
                      ├─→ response text
                      ├─→ confidence score
                      ├─→ intent detected
                      ├─→ source (kb/llm/support)
                      └─→ ticket_id (if escalated)
                          │
                          └──→ [Frontend]
                              └──→ Display to user
                                  ├─→ Show response
                                  ├─→ Show confidence badge
                                  ├─→ Show source label
                                  └─→ Update statistics
